## Что реализовано

- `GET /`
  - в браузере возвращает HTML-страницу (задание 1.2);
  - для API-клиента возвращает JSON `{"message": "Добро пожаловать в моё приложение FastAPI!"}` (задание 1.1).
- `POST /calculate` - считает сумму двух чисел.
- `GET /users` - возвращает данные пользователя из модели `User`.
- `POST /user` - возвращает данные пользователя и поле `is_adult`.
- `POST /feedback` - валидирует, сохраняет отзыв и возвращает сообщение об успешном сохранении.

## Почему `/` работает по-разному

В заданиях 1.1 и 1.2 для одного и того же маршрута `/` указаны разные требования (JSON и HTML).
Чтобы сдать всё в одном проекте, реализовано простое поведение:

- если запрос идёт из браузера - отдаётся `index.html`;
- если запрос идёт как API-запрос - возвращается JSON.

Так проект закрывает оба пункта без конфликта маршрутов.

## Файлы проекта

- `app.py` - основная логика приложения;
- `main.py` - точка входа для запуска командой из задания;
- `models.py` - Pydantic-модели;
- `index.html` - HTML-страница для задания 1.2;
- `check_app.py` - быстрая самопроверка маршрутов.

## Установка и запуск

### Вариант 1 (как в задании)

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux / macOS:

```bash
source .venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Запуск:

```bash
uvicorn main:app --reload
```

### Вариант 2 (показывает, что переменная приложения переименована)

```bash
uvicorn app:api --reload
```

## Что открыть в браузере

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/docs`

## Быстрая проверка без ручного тестирования

```bash
python check_app.py
```

Если всё в порядке, скрипт выведет:

```bash
Все проверки пройдены.
```
